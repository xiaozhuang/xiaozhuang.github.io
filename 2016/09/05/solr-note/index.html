Solr Note

发表于 2016-09-05   |   分类于 Note

Solr是一个独立的企业级搜索应用服务器，它对外提供类似于Web-service的API接口。用户可以通过http请求，向搜索引擎服务器提交一定格式的XML文件，生成索引；也可以通过Http Get操作提出查找请求，并得到XML格式的返回结果。

安装

windows

windows:解压solr-5.5.2.zip，cmd进入bin目录

安装windows服务：https://www.norconex.com/how-to-run-solr5-as-a-service-on-windows/

Centos

http://idroot.net/tutorials/how-to-install-apache-solr-on-centos-7/

启动

访问：http://localhost:8983/solr/
进入管理界面

Create a Core

Solr里面的core就像数据库里的表，用来管理索引和相关配置

Ask Questions

全文检索（searches all document fields for “video”）

搜索关键字，默认不区分大小写

返回json：http://10.10.60.61:8983/solr/techproducts/select?q=video&wt=json&indent=true
返回xml：http://10.10.60.61:8983/solr/techproducts/select?q=utf-8&wt=xml&indent=true

Solr can output the results in JSON, PHP, Ruby and even user-defined formats

指定返回字段

http://localhost:8983/solr/gettingstarted/select?q=video&fl=id,name,price

only contain the ID, name, and price for each returned document.

指定搜索字段

http://localhost:8983/solr/gettingstarted/select?q=name:black

searches for “black” in the name field only

范围条件查询

http://localhost:8983/solr/gettingstarted/select?q=price:[0 TO 400]&fl=id,name,price

facet

Facet是Solr的高级搜索功能之一，Solr作者给出的定义是导航(Guided Navigation)、参数化查询(Paramatic Search)。Facet的主要好处是在搜索的同时，可以按照Facet条件进行分组统计，给出导航信息，改善搜索体验。

它并不会修改查询结果信息，只是在查询结果上根据分类添加了count信息， 然后用户根据count信息做进一步的查询

http://10.10.60.51:8983/solr/techproducts/select?q=price:[0%20TO%20400]&fl=id,name ,price&facet=true&facet.field=cat

检索条件

q：搜索字段，字段和值冒号分隔，支持or和and

fq：多个不同字段查询，默认and操作

sort：排序，asc升序

start，rows：分页

fl：字段field

配置文件

Per Solr Core:

core.properties defines specific properties for each core such as its name, the collection the
core belongs to, the location of the schema, and other parameters. For more details on core.pro
perties, see the section Defining core.properties.

solrconfig.xml controls high-level behavior. You can, for example, specify an alternate location
for the data directory. For more information on solrconfig.xml, see Configuring solrconfig.xml.

schema.xml (or managed-schema instead) describes the documents you will ask Solr to index.
Inside schema.xml, you define a document as a collection of fields. You get to define both the
field types and the fields themselves. Field type definitions are powerful and include information
about how Solr processes incoming field values and query values. For more information on schem
a.xml, see Documents, Fields, and Schema Design. If you are using Solr’s Schema API to
manage your fields, you would see managed-schema instead of schema.xml (see Managed
Schema Definition in SolrConfig for more information).

data/ The directory containing the low level index files.
