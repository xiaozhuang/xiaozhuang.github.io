IntelliJ IDEA使用笔记

发表于 2016-12-10   |   分类于 Note

IDEA 全称 IntelliJ IDEA，是java语言开发的集成环境，IntelliJ在业界被公认为最好的java开发工具之一，尤其在智能代码助手、代码自动提示、重构、J2EE支持、各类版本工具(git、svn、github等)、JUnit、CVS整合、代码分析、 创新的GUI设计等方面的功能可以说是超常的。

快捷键

  • Alt + Insert 生成getter和setter
  • Ctrl + O 覆盖基类方法
  • Ctrl + Alt + O 移除未使用引用
  • Alt + Enter 快速修正，创建接口实现类
  • Ctrl + Shift + U 大小写转换
  • Ctrl + Shift + Alt + L 格式化代码
  • Ctrl + Space 基本的代码提示
  • Ctrl + Shift + Space 智能提示
  • Ctrl + Shift + Enter 自动补全末尾的字符
  • Shift + Enter 光标换行
  • Ctrl + N 快速打开类
  • Ctrl + Shift + N 快速打开文件
  • Ctrl + Shift + Alt + N 快速打开方法
  • Ctrl + H 打开类继承关系
  • Ctrl + Alt + H 查找方法使用

  • Shift + Shift 快速查找

  • Ctrl + Alt + T 选中代码，添加try catch

  • Ctrl + Alt + L 格式化代码
  • Ctrl + Shift + T 创建单元测试
  • Ctrl + Shift + F10 测试当前方法

  • Ctrl+F12 显示当前文件的结构

  • Shift + Esc 隐藏项目导航

  • Alt + Esc 显示项目导航

  • Ctrl + D 复制当前行

  • Ctrl + Y 删除当前行

  • Ctrl + E 最近的文件

  • Ctrl + Alt 点击类方法，打开方法实现

  • Ctrl + Shift + A 查找快捷键

  • Alt + 数字 打开侧边栏

  • Alt + Shift + [ 项目切换

  • F11 添加书签

  • Shift + F11 查看书签

  • Alt + Shift + F 添加收藏

  • Ctrl+Alt+S 插件启停

  • Ctrl + Shift + Alt + J 编辑相同内容

  • Alt + Insert 在测试类名上执行，添加测试方法

  • sout –> System.out.println();

  • soutp –> System.out.println(“”);
  • soutv –> System.out.println(“变量名 = “ + 变量);
  • soutm –> System.out.println(“当前类名.当前方法”);
  • psvm –> public static void main(String[] args) {}

调试

  • F7 单步进入
  • F8 下一步yi
  • Shift + F8 跳至下一个断点

重构

  • Shift + F6 重命名

  • Ctrl + F6 方法重构

  • Ctrl + Shift + Alt + T 重构

  • Ctrl + Alt + V 提取变量
  • Ctrl + Alt + M 方法提取

Git

右键–>git–>annotate idea中查看java文件修改历史

智能提示

代码生成

常用的有fori/sout/psvm+Tab即可生成循环、System.out、main方法等boilerplate样板代码
用Ctrl+J可以查看所有模板

后缀自动补全功能(Postfix Completion)，比模板生成更加灵活和强大。
for(User user : users) –> user.for + Tab
Date birthday = user.getBirthday(); –> user.getBirthday().var + Tab
user.nn –> if(user != null)

configure Spring facet in IntelliJ IDEA

Go to File/Project Structure/Modules, click the green plus icon, select Spring from the dropdown and select your module in the next dialog.
Then click the green plus in the right pane, click plus and select your Spring configuration files and classes and click OK.

取消重启时从新打开项目

File->settings->system Settings-> Reopen last project on startup 取消勾选

Tomcat VM options

-Xms512m -Xmx1024m -XX:PermSize=256M -XX:MaxPermSize=512M -Dfile.encoding=UTF-8

-Xmx2048m -Xms2048m -Xmn512m -XX:MaxPermSize=512m

idea 把一个add到git的文件去掉

Git -> Repository -> Reset HEAD…

idea @Override is not allowed when implementing interface method

  • 在project/module上，右键选择菜单“Open Module Settings” ;

  • 选择Modules，修改“Language Level” 为 “8-Lambdas,……”;

  • 选择Project， 修改“Project language level” 为 “SDK default”。

JDK配置

File –>Project Structure -> sdks

idea乱码

Settings → File Encoding → Project Encoding → IDE Encoding.

IntelliJ IDEA Tomcat Debug启动非常慢，系统运行非常慢

看看你在debug的时候是否在方法上加了断点，如果在方法上加了断点，那么运行时非常慢。

远程调试

添加remote tomcat
远程服务IP，端口8086

Sartup/Connection -> Debug
-agentlib:jdwp=transport=dt_socket,address=8086,suspend=n,server=y

服务端：JAVA_OPTS中添加
-agentlib:jdwp=transport=dt_socket,address=8086,suspend=n,server=y

maven本地jar包在idea中dependencies总是有红线解决办法

对于有波浪线的dependency，将其从pom中删除，保存后，再撤销回来，然后reimport
就ok了

Idea编译无法输出profile资源文件问题

由于idea使用的编译器是Javac（在settings—Java Compiler里配置）

Intellij IDEA 自动生成 serialVersionUID

启用 校验 Serializable 校验

Setting->Inspections->Serialization issues->Serializable class without ’serialVersionUID’
选上以后，在你的class中：Alt+Enter就会提示自动创建serialVersionUID了。

默认配置
Other Settings -> Settings for New Projects->Serialization issues->Serializable class without ’serialVersionUID’

错误解决

Unable to open debugger port

删除这个 Tomcat 配置，让 Intellij IDEA 自动找debug新的端口
查看debug端口，tomcat配置 Startup/Connection -> Debug

显示侧边栏

view -> tool Buttons

Maven Projects模块灰色

在idea中maven的setting中找到ignored files，取消被忽略的模块

maven项目不包含在root pom中模块加载

maven工程中不包含在root pom modules配置中的模块，idea不自动加载依赖
idea Maven面板中也没有该模块，也就不能执行打包

解决方式:
mavan面板中，点击“加好” 选择该模块pom.xml
