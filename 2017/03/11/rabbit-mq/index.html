RabbitMQ

发表于 2017-03-11   |   分类于 Note

RabbitMQ是实现了AMQP标准的代理服务器，RabbitMQ运行集群简单可靠。

RabbitMQ简介

  • 除了Qpid之外，RabbitMQ是唯一实现了AMQP标准的代理服务器
  • 由于RabbitMQ使用Erlang开发，RabbitMQ集群不可思议的简单
  • RabbitMQ可靠

安装

rabbitmq-server

记得防火墙开启端口，默认：5672
默认用户：guest，默认密码：guest

Management插件

Python驱动

查看RabbitMQ 命令行管理管理工具rabbitmqctl

消息通信

AMQP消息能以一对多的广播方式进行路由，也可以以一对一的方式路由

消息包含两部分内容：有效负荷（payload）和标签（label）

有效负荷是要传输的数据；标签描述了有效负荷。

生产者创建消息并设置标签。
消费者接收消息时，只得到消息的一部分，有效负荷。
如果消费者需要明确知道生产者信息，生产者需将发送方信息放入有效负荷中。

信道

应用程序和RabbitMQ代理服务器之间创建一条TCP连接。
线程启动后，会在现成的连接上创建一条信道，可就获得了连接到RabbitMQ上的私密通信路径。
所有线程只使用一条TCP连接以满足性能方面的要求，又能保证每个线程的私密性。

在一条TCP连接上创建多少条信道是没有限制的。
