Think In Java

Think In Java 学习笔记

一切都是对象

Java中（几乎）一切都是对象

存储到什么地方

栈：位于RAM中，存储对象引用和基本类型数据
堆：一种通用的内存池，位于RAM中，存放所有的java对象。
常量存储：常量值通常直接存放在程序代码内部。

基本类型

基本类型不用new创建变量，而是创建一个并非是引用的“自动”变量，变量直接存储“值”
并置与栈中。

数值类型都有正负号

字段和方法

基本数据类型默认值
基本数据类型成员变量有默认值，局部变量没有默认值
对象引用默认为null

类型 默认值
boolean false
char 0
byte 0
short 0
int 0
long 0L
float 0.0f
double 0.0d

参数列表

基本类型，值保存在变量中。
引用类型，变量中保存的是实际对象的地址。

图片来源：java到底是值传递还是引用传递

java中方法参数传递方式是按值传递。
如果参数是基本类型，传递的是字面量值的拷贝。
如果参数是引用类型，传递的是引用的拷贝。

初始化和清除

方法重载

独一无二的形参类型列表
顺序不同也可以，但会产生难以维护的代码
不能根据返回值类型区分方法重载

this关键字

在方法内部，可用this获得当前对象的句柄。

可在一个构建器中调用另一个构建器，使用this实现。

finalize() 用途何在

如果对象分配了一个特殊内存区域，没有使用new
垃圾收集器只释放那些由new分配的内存

之所以使用finalize()，是因为有时需要采用与java的普通方法不同的方法，
通过分配内存做一些具有C风格的事情(通过本地方法实现)

你必须实施清理

无论是垃圾回收还是终结,都不保证一定会发生。
如果JVM并未面临内存耗尽的情形，它不会浪费时间去执行垃圾回收以恢复内存。

垃圾收集器如何工作

引用计数

引用计数是一种简单但速度很慢的垃圾回收技术。
管理引用计数开销很大，且如果对象之间存在循环引用，
会出现“对象应该被回收，但引用计数不为0”的情况。

引用计数常用来说明垃圾收集的工作方式，
但似乎从未被应用于任何一种Java虚拟机实现中。

可达性分析

对于任何“活”的对象，一定能最终追溯到其存活在堆栈或静态存储区中的引用

垃圾回收器自适应

“自适应的、分代的、停止-复制、标记-清扫”式垃圾回收器。

即时编译器技术(Just-In-Time,JIT)

这种技术可以把程序全部或部分翻译成本地机器码。

两种方案：

  • 即时编译器编译所有代码（增加可执行代码的长度，导致页面调度，降低程序速度）
  • 惰性评估(即时编译器只在必要时编译代码)

成员初始化

静态初始化只有在必要时刻才会进行

  • 创建对象前
  • 引用静态变量

Java类实例化执行顺序：

  • 静态声明
  • 静态初始化块
  • 成员声明
  • 实例初始化块
  • 构造函数

操作符

逻辑运算符

AND ( && )
OR ( || )
NOT ( ! )

只能将AND、OR、NOT应用于布尔值

按位运算

  • 位OR | 或
  • 位AND & 与
  • 位NOT ~ 非
  • 位XOR ^ 异或

两值不同，异或结果为真

移位运算

char、byte、short 进行移位处理，在移位前，会自动转换为int

Tools

对象同名属性拷贝
