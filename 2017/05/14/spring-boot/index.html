Spring Boot

发表于 2017-05-14   |   分类于 Note

Spring Boot是由Pivotal团队提供的全新框架，其设计目的是用来简化新Spring应用的初始搭建以及开发过程。
该框架使用了特定的方式来进行配置，从而使开发人员不再需要定义样板化的配置

概述

4个核心

  • 自动配置 为常见配置常见进行自动化配置；如ClassPath里发现H2数据库的库，自动配置H2数据库
  • 起步依赖 把库聚合再一起，并且依赖库的版本已经过测试，不会出现不兼容
  • 命令行界面 Spring Boot Cli 让只写代码即可实现应用程序成为可能
  • Actuator 提供运行时检视应用内部情况的能力

Actuator 包含

  • Spring上下文配置的Bean
  • Spring Boot自动配置做的决策
  • 应用取到的环境变量、系统变量、配置属性和命令行参数
  • 最近处理过的HTTP请求的追踪
  • 内存用量、垃圾回收、web请求及数据源用量相关指标

Spring Boot配置

ReadingListApplication.java : 应用程序的启动引导类（boot class），也是主要的Spring配置类
application.properties : 用于配置应用程序的和Spring Boot的属性

@SpringBootApplication
开启了Spring的组件扫描和Spring Boot的自动配置功能；
实际是3个注解的组合：

  • @Configuration： 标明该类使用Spring基于Java的配置
  • @ComponentScan： 启用组件扫描(on the package where the application is located)
  • @EnableAutoConfiguration： 开启Spring Boot自动配置

如果需要Spring Boot自动配置以外的其他Spring配置， 可配置到@Configuration标注的类里

application.properties

停用AutoConfig

由@SpringBootApplication 改为@Configuration，@ComponentScan

Spring Boot maven插件

Spring Boot maven 构建插件主要功能是把项目打包成一个可执行的超级JAR（uber-JAR）
将依赖打入JAR，并添加描述文件，使可通过java -jar来运行

spring-boot-starter-parent

将spring-boot-starter-parent作为模块parent，
继承了很多常用库的依赖版本，声明依赖时，不用再指定版本号

起步依赖

起步依赖ArtifactId以spring-boot-starter-前缀

起步依赖将常用依赖进行分组整合，降低项目依赖的复杂度

起步依赖并不需要指定版本号，起步依赖的版本是由正在使用的Spring Boot的版本决定的

起步依赖中的各个依赖版本之间的兼容性都经过测试，只在特殊情况下覆盖这些依赖。

Spring Boot条件化配置

org.springframework.boot.autoconfigure

spring-boot-autoconfigure的JAR文件，包含很多条件化配置类

SpringBoot 运用Spring条件化配置，定义多个特殊的条件化配置注解

  • @ConditionalOnBean 配置了某个Bean

  • @ConditionalonMissBean 没有配置某个Bean

  • @ConditionalOnClass ClassPath里有指定的类

  • @ConditionalOnMissingClass ClassPath里缺少指定的类

  • @ConditionalOnExpression 给定的Spring Expression Language（SpEL）表达式计算结果为true

  • @ConditionalOnJava Java的版本匹配特定值或者一个范围

  • @ConditionalOnJndi 参数中给定的JNDI必须存在一个；如果没有参数，则要有JNDI InitialContext

  • @ConditionalOnProperty 指定配置属性要有一个明确的值

  • @ConditionalOnResource ClassPath里有指定的资源

  • @ConditionalOnWebApplication 是一个Web应用

  • @ConditionalOnNotWebApplication 不是一个Web应用

@EnableConfigurationProperties
{@link ConfigurationProperties} beans can be registered in the standard way

自定义配置

覆盖Spring Boot自动化配置

要覆盖Spring Boot的自动配置，只需编写一个显式的配置。
Spring Boot的设计是加载应用级的配置，随后在考虑自动配置。

通过属性文件外置配置

Spring Boot自动配置的Bean提供300多个微调的属性

SpringBoot 支持多种设置途径（按优先级顺序）

  • 命令行参数
  • java:comp/env里的JNDI属性
  • JVM系统属性
  • 操作系统环境变量
  • 随机生成的带random.* 前缀的属性
  • 应用程序外的application.properties 或application.yml
  • 应用程序内的application.properties 或application.yml
  • 通过@PropertySource标注的属性源
  • 默认属性

application.properties、application.yml文件能放在4个位置（按优先级顺序）

  • 外置，相对于应用程序目录的/config 子目录里
  • 外置，相对应用程序运行的目录里
  • 内置，config包里
  • 内置，ClassPath根目录

深入Actuator

Spring Boot的Actuator监控和度量应用程序的端点

Actuator端点

  • autoconfig 自动配置报告（记录哪些自动配置条件通过）
  • configprops 配置属性如何注入Bean
  • beans 描述应用程序上下文里的全部Bean
  • dump 线程活动的快照
  • env 全部环境属性
  • env/{name}
  • health 应用健康指标（由HealthIndicator实现类提供）
  • info 应用程序定制信息
  • mappings 全部URL路径及对应控制器
  • metrics 各种应用程序度量信息，内存用量和HTTP请求计数
  • metrics/{name} 指定名称应用程序度量信息
  • trace 基本的HTTP请求跟踪信息（时间戳、HTTP头等）
  • shutdown 关闭应用程序，要求endpoints.shutdown.enabled=true (post)

spring-boot-starter-actuator安全使用规范
