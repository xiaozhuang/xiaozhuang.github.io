cli

数据绑定

实例

el 用于指定页面中已经存在的DOM元素，来挂载Vue实例；可通过app.$el访问该元素。

Vue实例本身也代理了 data 对象的所有属性。

也可以指定一个已有的变量，默认建立双向绑定。

生命周期

  • created 实例创建完成后，未挂载
  • mounted el挂载之后
  • beforeDestorey 实例销毁之前

插值与表达式

Mustache语法 将双向绑定的数据显示出来。

过滤器

使用管道符|对数据进行过滤

语法糖

v-bind 可简写为 :xxx
v-on 简写为 @xxx

计算属性

计算属性以函数形式 定义在 computed 中

计算属性经常用于动态设置元素 class、style，动态传递 props

两个技巧：

  • 可依赖其他计算属性
  • 可依赖其他实例的数据

计算属性缓存

与methods中定义方法不同， 计算属性是基于它的依赖 缓存的； methods则不同，每次重新渲染，都被调用。

v-bind及class与style绑定

class绑定

:class 等同于 v-bind:class

style绑定

:style 等同于 v-bind:style

内置指令

指令

指令的主要职责是 当其表达式的值改变时，相应的行为应用到DOM上。

数据驱动DOM是Vue的核心概念。

v-bind: 单向绑定，数据变化会更新dom
v-model: 双向绑定

Vue实例 将methods里的方法也代理了;

基本指令

directives

v-cloak

v-cloak 不需表达式，Vue结束编译时，从绑定的HTML元素上移除。

简单项目中，v-cloak是解决初始化导致页面闪动的最佳实践。
使用vue-router时，项目HTML结构只是一个空的div元素，剩余的内容都是有路由挂载不同组件完成的，所以不再需要v-cloak;

v-once

作用于它的元素或组件，只渲染一次； 渲染后不随数据变化重新渲染。

组件

props 数据传递

inheritAttrs

$refs 访问子组件实例或子元素

ref 提供对内部指定元素的访问

mixins

混入对象的钩子将在组件自身钩子之前调用

值为对象的选项，例如 methods, components 和 directives，将被混合为同一个对象。
